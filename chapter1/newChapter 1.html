2.1IP addresses:

An IP (Internet Protocol) address is a unique numerical label assigned to devices connected to a network that uses the Internet Protocol for communication. It serves two main functions: identifying the host or network interface and providing the location of the host in the network.

2.1.1Private IP address:

In your local network, each device (phone, laptop, printer, etc.) gets a private IP address. These addresses are used within your home network only and are not visible to the public internet. They allow the devices in your home to communicate with each other and with the router.

How it works

  • Your router is acting as the central hub of your local network (called a LAN or Local Area Network).

  • The router assigns private IP addresses to each device connected to it, usually through a service called DHCP (Dynamic Host Configuration Protocol), which automatically assigns IP addresses.

2.1.2Public IP address:

This is the IP address that your router uses to communicate with the global internet. It is assigned by your Internet Service Provider (ISP) (like Airtel, Jio, BSNL) , and it is unique across the entire internet. All your devices share this public IP address when accessing the internet, thanks to a process called Network Address Translation (NAT), where your router translates private IPs into the public IP when your devices communicate with external servers.

2.1.3Dynamic IP addresses:

Dynamic IPs are assigned by the network dynamically when a device connects. These addresses can change over time.

Your router will be assigned a public IP address by your ISP, which allows your home network to be connected to the wider internet.

This is the IP address that websites and external services see when you visit them from your home. However, each device inside your local network will have its own private IP, and only the router's public IP will be visible to the internet.

2.1.4Dynamic IP vs. Static IP:

Most home users are assigned a dynamic public IP address by their ISP. This means the public IP address assigned to your router can change periodically, for example, every time you reboot the router or after a certain time period set by your ISP.

A static public IP is an IP address that does not change and is permanently assigned to your router. Businesses or individuals who need consistent access (e.g., hosting a server, using remote access tools) often request static IPs from their ISP.

2.2Physical / Mac addresses.

A MAC (Media Access Control) address is a unique hardware identifier assigned to each network interface on your devices. This is a permanent, physical address and is different for each network interface (e.g., your phone's Wi-Fi adapter and your laptop's Wi-Fi adapter will each have their own MAC address).

How it works :

  • Every device connected to your router has a MAC address. For instance:

  • MAC addresses are used for local network communication. When your laptop sends data to your phone over Wi-Fi, it uses the MAC addresses of both devices to ensure the data reaches the correct hardware.

  • The router also uses MAC addresses internally to keep track of which device is assigned which private IP address. For example:

    • Laptop MAC 00:1A:2B:3C:4D:5F ↔ Private IP 192.168.1.10

    • Phone MAC 00:1A:2B:3C:4D:5E ↔ Private IP 192.168.1.11

The MAC address stays within the local network. It is not used when communicating over the internet (global network). On the internet, only your public IP is used to identify your router.

Putting It All Together (Example Scenario):

  1. At Home (Local Network):

    • Each device (laptop, phone, printer) in your home has:

      • A private IP (e.g., laptop: 192.168.1.10, phone: 192.168.1.11, printer: 192.168.1.12).

      • A MAC address (e.g., laptop: 00:1A:2B:3C:4D:5F, phone: 00:1A:2B:3C:4D:5E), used for communication inside the home network.

    • The router manages the local network and assigns private IPs to each device, mapping them to their respective MAC addresses.

  2. Accessing the Internet (Global Network):

    • When your phone or laptop accesses a website, the router uses NAT to translate the private IP of the device (e.g., 192.168.1.11) into the router's public IP (e.g., 103.48.112.150).

    • The website you visit only sees your public IP (e.g., 103.48.112.150), not your device's private IP or MAC address.

    • If your ISP has assigned a dynamic IP, this public IP might change from time to time. If it's a static IP, it remains the same.

2.3Ports and Sockets

2.3.1Ports

A port address is a 16-bit number that uniquely identifies a specific process or service running on a machine within a network.

Ports allow a device (with a specific IP address) to differentiate between multiple services and applications.

In other words, a port tells the network where to send data within a particular device.

How Port Addresses Work:

    1. You open a web browser and enter a URL like www.example.com.

    2. Your web browser (the client) needs to connect to the web server hosting that website (the server).

    3. The browser sends a request to the server's IP address, targeting port 80 (for HTTP traffic) or port 443 (for HTTPS traffic).

    4. The operating system on your device assigns a dynamic port (ephemeral port) to your browser for this session (e.g., port 52000).

    5. The server receives the request on its public IP and port (e.g., IP 203.0.113.5, port 80), processes the request, and sends back the response to your browser at your IP address and ephemeral port (e.g., 192.168.1.10:52000).

1Sockets

In network communication, both IP addresses and port numbers are required to ensure data is sent to the correct destination , This combination of an IP address and a port number is called a socket.

Socket Address example:

    203.0.113.5:80 (where 203.0.113.5 is the IP address and 80 is the port number).

How Port and IP Address are Used in Packets:

When data is transmitted over the internet, it is broken into packets. Each packet has several layers of information. Two of the most important layers in the context of port and IP addresses are:

  1. IP Layer (Network Layer): Contains the source and destination IP addresses, which are used to route the packet between devices (across networks).

    • Source IP Address: The IP address of the sending device.

    • Destination IP Address: The IP address of the receiving device.

  2. Transport Layer: This layer adds the source and destination port numbers to the packet. These port numbers are used to direct the data to the correct process on the sending and receiving devices.

    • Source Port: The port number used by the sending process (e.g., a web browser with ephemeral port 52000).

    • Destination Port: The port number used by the receiving process (e.g., a web server listening on port 80 or 443).

When a packet is sent over the internet:

  • Source IP and Source Port: Identify the sender (e.g., your device 192.168.1.10:52000).

  • Destination IP and Destination Port: Identify the recipient (e.g., a web server 203.0.113.5:80).

Packet Example:

Suppose your browser sends a request to visit www.example.com:

  • Source IP: 192.168.1.10 (your device's private IP on your local network).

  • Source Port: 52000 (ephemeral port assigned by your operating system).

  • Destination IP: 203.0.113.5 (the public IP of the web server hosting www.example.com).

  • Destination Port: 80 (the well-known port for HTTP traffic).

The packet will look something like this:

  • IP Header: Contains 192.168.1.10 (source IP) and 203.0.113.5 (destination IP).

  • Transport Layer (TCP/UDP): Contains 52000 (source port) and 80 (destination port).

When the server responds, the roles are reversed:

  • Source IP: 203.0.113.5 (the web server).

  • Source Port: 80.

  • Destination IP: 192.168.1.10 (your device).

  • Destination Port: 52000 (your browser's ephemeral port).

Why are Port Addresses Necessary?

Without ports, a device could only run one service at a time on the same IP address. Ports allow a device to run multiple services (web, email, file sharing, etc.) simultaneously and direct traffic to the correct application.

3.2.1Virtual Packet Switching

Virtual packet switching, commonly referred to as virtual circuit switching, is a hybrid between traditional packet switching and circuit switching. It creates a temporary, logical path (or “virtual circuit") through a network for the duration of a communication session, providing some benefits of both circuit and packet switching.

Key Characteristics of Virtual Circuit Switching:

  1. Connection-Oriented:

    • Before data transmission begins, a logical path (called a virtual circuit) is established between the sender and receiver. This path remains in place for the entire communication session.

    • The virtual circuit ensures that all packets follow the same route, unlike standard packet switching where each packet can take different routes.

  2. Packet-Based:

    • Even though a virtual circuit is established, data is still broken down into packets, like in packet switching.

    • However, since the virtual circuit ensures a consistent route, the order of the packets is maintained, and reordering is usually unnecessary.

  3. Temporary:

    • The virtual circuit is not permanent. It is created only for the duration of the communication session and is torn down afterward.

  4. Efficient and Reliable:

    • Virtual circuit switching combines the efficiency of packet switching (sharing network resources) with the reliability of circuit switching (a predefined path for the session).

    • It also provides some form of quality of service (QoS) by maintaining a dedicated logical path for the data.

3.3Circuit Switching

In circuit-switched networks, resources like transmission rate and buffers are reserved for the entire duration of a communication session between two end systems, similar to making a reservation at a restaurant. Traditional telephone networks are an example, where a dedicated end-to-end connection (or circuit) is established before data transfer, ensuring a guaranteed constant transmission rate.

In contrast, packet-switched networks (such as the Internet) do not reserve resources. Data is sent as packets without guaranteed access to transmission resources, meaning packets may experience delays if links are congested. The Internet operates on a “best effort” basis, trying to deliver packets as quickly as possible without guarantees.

3.4Delay and Losses.

In computer networking, several types of delays affect how packets traverse from one router to another. Understanding these delays is crucial for optimizing network performance. Here's a summary of the different delays:

  1. Processing Delay:

    • Definition: The time required by a router to process a packet's header, check for errors, and determine where to forward the packet.

    • Typical Duration: Usually in microseconds or less in high-speed routers.

    • Impact: Generally minor but can affect a router's throughput.

  2. Queuing Delay:

    • Definition: The time a packet spends waiting in a queue before it can be transmitted onto the link.

    • Factors: Depends on the number of packets ahead in the queue and network traffic intensity.

    • Typical Duration: Can range from microseconds to milliseconds.

    • Impact: Can vary significantly based on network load and traffic patterns.

  3. Transmission Delay:

    • Definition: The time required to push all bits of a packet into the link. Calculated as the packet length (L) divided by the link's transmission rate (R).

    • Example: For a 10 Mbps link, it takes longer than for a 100 Mbps link to transmit the same packet.

    • Typical Duration: Typically in microseconds to milliseconds.

    • Impact: Dependent on packet size and link speed.

  4. Propagation Delay:

    • Definition: The time it takes for a bit to travel from one router to the next, based on the distance between them and the speed of the signal through the medium.

    • Calculation: Distance (d) divided by the propagation speed (s) of the link.

    • Typical Duration: Ranges from microseconds to milliseconds, but can be longer in wide-area networks.

    • Impact: Dependent on distance and physical medium of the link.

Comparing Transmission and Propagation Delay

  • Transmission Delay: Affects how quickly a router can push packets onto the link, independent of distance. It depends on packet length and link speed.

  • Propagation Delay: Affects how quickly a signal travels over a distance, independent of packet size or link speed.

3.5.6ARP AND RARP

ARP(Address Resolution Protocol) :

ARP is used to find the physical (MAC) address associated with a device's IP address on a local network.

Scenario: Suppose a computer knows the IP address of another device it wants to communicate with, but it needs to know the MAC address (which is used for actual data transmission on the local network).

Example 2. Working

  • A device sends out an ARP request to all devices in the network (a broadcast) asking, “Who has this IP address?”

  • The device with that IP address responds with its MAC address (an ARP reply).

  • The requesting device then stores the IP-to-MAC address mapping for future use in its ARP cache.

RARP (Reverse Address Resolution Protocol):

RARP is used to find the IP address associated with a device's MAC address.

Scenario: RARP is typically used by diskless machines when they first boot up. These machines know their own MAC address but don't know their IP address, so they use RARP to request it.

Steps in the Three-Way Handshake:

  1. SYN (Synchronize):

    • The client sends a TCP segment with the SYN flag set to the server. This is the initial request to establish a connection.

    • It includes an initial sequence number (let's say x), which will be used to track data sent by the client.

  2. SYN-ACK (Synchronize-Acknowledge):

    • The server receives the SYN request from the client and responds with a TCP segment that has both the SYN and ACK flags set.

    • The ACK acknowledges the client's SYN request (acknowledging x+1).

    • The SYN part of this message includes the server's initial sequence number (let's say y), which will be used to track data sent by the server.

  3. ACK (Acknowledge):

    • The client receives the SYN-ACK from the server and sends an ACK back, acknowledging the server's sequence number (acknowledging y+1).

    • Now, the connection is fully established, and data transmission can begin.

Question 2. Since udp connection is an unreliable way to tranfer pacekts and so in packet swithching , Is packet switching a udp connection ?

Solution. Packet switching is a method of data transmission, but it is not specifically tied to either UDP (User Datagram Protocol) or TCP (Transmission Control Protocol). Both UDP and TCP use packet switching, but in different ways.

What is Packet Switching?

  • Packet switching involves breaking down data into small packets and sending them independently over a network. Each packet may take a different path to reach the destination, and at the destination, the packets are reassembled into the original message.

  • This approach maximizes network efficiency by allowing multiple communications to share the same network paths.

How UDP and TCP Use Packet Switching:

  • UDP (User Datagram Protocol):

    • UDP uses packet switching without establishing a connection first. Each packet is sent independently, without error checking or retransmission if a packet is lost.

    • Since there's no need to maintain a connection, UDP is faster but less reliable. It doesn't ensure that packets arrive in order or even at all.

    • Example: Live streaming or online gaming, where speed is prioritized over reliability, often uses UDP.

  • TCP (Transmission Control Protocol):

    • TCP also uses packet switching, but it adds reliability. Before transmitting, TCP establishes a connection (using the three-way handshake), ensures that packets arrive in order, checks for errors, and resends lost packets.

    • This connection-oriented nature makes TCP slower but highly reliable.

    • Example: File transfers or web browsing, where data accuracy and completeness are critical, typically use TCP.

Circuit Switching vs. Packet Switching:

  • Circuit Switching:

    • Establishes a continuous connection.

    • Used traditionally in voice communication, like telephone calls.

    • Data is not divided into packets; instead, it is sent as a continuous stream.

    • Ideal for real-time communication since the path guarantees consistent, uninterrupted data flow.

  • Packet Switching:

    • Breaks data into small packets, each transmitted independently and possibly taking different routes.

    • Used in computer networks, like the internet, where resource efficiency is more important than a guaranteed path.

    • Data may experience slight delays or arrive out of order but is reassembled at the destination.

Question 4. If TCP connection is reliable , is it possible that we don't suffer a packet loss?

Solution. No, even though TCP is reliable, packet loss can still occur due to network issues like congestion, faulty hardware, or interference. TCP mitigates this by detecting losses and retransmitting packets, ensuring data integrity despite potential losses.
